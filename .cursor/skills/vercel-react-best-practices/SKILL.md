# 技能定义页：React / Next.js 性能最佳实践

## 技能定位

这是一本面向 React / Next.js 工程任务的性能优化技能书。
它提供一套结构化规则，帮助在开发过程中更早识别性能风险，并在实现、审查和重构阶段采用更合适的方案。

## 技能目标

本技能主要解决以下问题：

- 页面或接口链路存在串行等待，导致响应变慢
- 首屏资源过大，影响加载与交互时间
- 服务端数据处理方式不合理，增加渲染成本
- 客户端订阅、状态管理或组件组织方式导致频繁重渲染
- 渲染阶段存在可避免的布局、绘制或执行开销
- JavaScript 运行时实现存在低效模式

## 何时使用

当任务包含以下内容时，建议使用本技能书：

- 新建 React 组件、Hooks、页面或布局
- 编写 Next.js 服务端逻辑、路由处理逻辑或数据获取逻辑
- 对已有页面做性能诊断与重构
- Review React / Next.js 性能实现
- 优化首屏加载、资源分发、缓存与渲染链路
- 处理"加载慢、包太大、重复渲染、交互卡顿"等问题

## 典型触发信号

如果任务描述中出现以下方向，通常适合参考本技能：

- "优化性能"
- "减少包大小"
- "修复请求瀑布"
- "减少重复渲染"
- "优化数据获取"
- "提升首屏速度"
- "重构组件结构"
- "优化 Next.js 页面加载表现"

## 技能范围

本技能覆盖以下八类主题：

| 优先级 | 主题 | 关注点 | 前缀 |
|---|---|---|---|
| 1 | 消除请求瀑布 | 并行化、延迟 await、拆解阻塞链路 | `async-` |
| 2 | 包体积优化 | 按需加载、动态导入、延迟非关键资源 | `bundle-` |
| 3 | 服务端性能优化 | 缓存、序列化控制、服务端并行获取 | `server-` |
| 4 | 客户端数据获取优化 | 请求去重、订阅与监听收敛 | `client-` |
| 5 | 重渲染优化 | 状态订阅粒度、依赖稳定性、记忆化 | `rerender-` |
| 6 | 渲染性能优化 | 显示策略、静态提升、渲染成本控制 | `rendering-` |
| 7 | JavaScript 运行时优化 | 循环、查找、缓存、提前退出 | `js-` |
| 8 | 进阶性能模式 | 稳定引用、复杂交互中的性能模式 | `advanced-` |

## 使用方式

### 快速使用
1. 根据问题类型定位主题分类
2. 在 `rules/` 中查找对应规则文件
3. 对照反例与正例判断现有实现
4. 结合上下文决定是否应用规则
5. 必要时回到完整手册查看整体原则

### 深入使用
如果任务涉及多个性能层面，建议先回到 `README.md` 理解整体结构，再进入 `rules/` 按主题逐项落地。
`AGENTS.md` 可作为全量规则汇编查看，但不作为本技能书的主入口。

## 建议输出方式

在实际使用本技能时，建议输出内容包含：

- 当前实现的性能风险点
- 建议采用的规则或章节
- 为什么该规则适用
- 修改后的预期收益
- 是否存在权衡或例外情况

## 不适用场景

以下情况不应机械套用本技能中的规则：

- 业务规模很小，优化收益远低于实现成本
- 代码路径并不在性能关键链路中
- 为了优化而显著损害可读性、可维护性或业务正确性
- 上下文不足，无法判断规则是否真正适用

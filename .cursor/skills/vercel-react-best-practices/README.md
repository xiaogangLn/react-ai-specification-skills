# 通义灵码技能书：React / Next.js 性能最佳实践

这是一本面向 React 与 Next.js 工程场景的性能优化技能书，用于在编码、评审、重构与性能治理过程中，快速调用一套可执行、可复用、可维护的最佳实践规则。

本仓库聚焦两类目标：

- 帮助开发者在日常迭代中减少常见性能问题
- 帮助在 React / Next.js 编码、审查与重构过程中采用更稳健的性能策略

## 仓库定位

本仓库不是普通的文档集合，而是围绕 React / Next.js 性能主题组织的一本技能书。
它将分散的经验规则整理为统一结构，便于：

- 编码时作为参考规范
- Code Review 时作为检查清单
- 重构时作为优化依据
- 沉淀为团队共识与辅助编码规范

## 适用场景

当你正在处理以下任务时，建议优先参考本技能书：

- 编写新的 React 组件或 Next.js 页面
- 设计客户端或服务端数据获取逻辑
- 排查加载慢、渲染慢、交互卡顿等性能问题
- 进行包体积治理、首屏性能优化、请求瀑布治理
- 审查现有实现是否存在重复渲染、过早加载、缓存缺失等问题
- 将经验规则沉淀为团队共识或辅助编码规范

## 核心主题

技能书中的规则围绕以下八类性能主题组织：

1. 消除请求瀑布
2. 包体积优化
3. 服务端性能优化
4. 客户端数据获取优化
5. 重渲染优化
6. 渲染性能优化
7. JavaScript 运行时优化
8. 进阶性能模式

这些主题覆盖从请求链路、服务端渲染、客户端状态订阅，到前端执行与渲染成本的主要性能问题。

## 仓库结构

- `README.md`：仓库总入口，说明技能书定位、结构与使用方式
- `SKILL.md`：技能定义页，描述技能用途、触发条件与适用边界
- `AGENTS.md`：完整规则汇编与兼容性产物，用于集中查看全量内容
- `metadata.json`：技能书元数据
- `rules/`：规则原文目录，每个文件对应一条实践
  - `_sections.md`：章节定义
  - `_template.md`：规则模板
  - `*.md`：具体规则文件

## 如何使用

### 作为性能参考手册
当你在写 React / Next.js 代码时，可以先查阅对应章节的规则文件，快速确认推荐写法与常见反例。

### 作为代码审查清单
在 Review 组件、页面、数据请求与交互逻辑时，可按主题逐项检查，例如：

- 是否存在串行请求导致的瀑布
- 是否把大模块过早打进主包
- 是否存在无意义的重渲染
- 是否把过多数据传给客户端组件
- 是否遗漏缓存、去重或延迟加载策略

### 作为重构依据
当页面性能、交互流畅度或资源加载表现不理想时，可根据规则定位问题类型，再决定优先级与落地方式。

## 推荐阅读路径

如果你是第一次使用本技能书，建议按以下顺序阅读：

1. 先看 `README.md`，了解技能书定位、结构与使用方式
2. 再看 `SKILL.md`，了解技能的适用范围、触发时机与使用边界
3. 最后进入 `rules/` 目录，按具体问题查阅单条规则

`AGENTS.md` 可作为全量规则汇编或兼容性产物查看，但不作为本技能书的主入口。

## 规则文件编写约定

每条规则建议保持统一结构，便于查阅与后续维护：

- 规则标题
- 为什么重要
- 适用场景
- 不适用场景
- 反例
- 正例
- 落地提示
- 参考资料

规则文件命名按主题前缀组织，例如：

- `async-`：消除请求瀑布
- `bundle-`：包体积优化
- `server-`：服务端性能优化
- `client-`：客户端数据获取优化
- `rerender-`：重渲染优化
- `rendering-`：渲染性能优化
- `js-`：JavaScript 运行时优化
- `advanced-`：进阶性能模式

## 维护建议

当你新增或修改规则时，建议遵循以下流程：

1. 在 `rules/` 中补充或更新规则文件
2. 确保反例与正例足够清晰
3. 明确规则适用条件与限制
4. 更新入口、定义页与完整手册，保持整体一致
